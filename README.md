## AI integration

I’m interested in AI as a product layer, not as decoration.

The best AI features should reduce friction, clarify decisions and fit naturally into the workflow. I like building AI-assisted interfaces that feel useful, controlled and understandable.

```txt
ai-integration/
├── prompt systems
├── workflow automation
├── internal copilots
├── document and data interfaces
├── decision support dashboards
├── content generation tools
└── human-in-the-loop UX
```

---

## Frontend principles

```txt
Good UI is not decoration.
It is structure, rhythm, feedback and restraint.

Good frontend is not just components.
It is contracts, state, performance, accessibility and long-term clarity.

Good AI integration does not replace product thinking.
It makes the right action easier to take.
```

---

## How I like to work

1. Understand the product problem.
2. Define the interface structure.
3. Map states, data and user decisions.
4. Build the frontend architecture.
5. Add AI only where it improves the workflow.
6. Polish interaction, performance and accessibility.
7. Document the decisions so the system can evolve.
